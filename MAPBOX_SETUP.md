## What You Need to Do

### 1. Get a Mapbox Access Token

1. Go to [Mapbox.com](https://www.mapbox.com)
2. Sign up for a free account (includes 200,000 free map loads/month)
3. Go to your [Account page](https://account.mapbox.com/)
4. Create a new access token with these scopes:
   - `styles:read`
   - `geocoding:read`
5. Copy your access token

### 2. Add the Token to Your File

Open `index-mapbox.html` and replace this line:

```javascript
mapboxgl.accessToken = 'YOUR_MAPBOX_ACCESS_TOKEN';
```

With your actual token:

```javascript
mapboxgl.accessToken = 'pk.eyJ1IjoieW91cnVzZXJuYW1lIiwiYSI6ImFiYzEyM...';
```

### 3. Commit and Push

```bash
cd ~/Documents/GitHub/hq-outages
git add index-mapbox.html index-landing.html .github/workflows/deploy-mapbox.yml MAPBOX_SETUP.md
git commit -m "Add Mapbox visualization with separate workflow"
git push
```

The separate Mapbox workflow will run automatically after the main Leaflet workflow completes.

## How the Separate Workflow Works

The new `deploy-mapbox.yml` workflow:

1. **Triggers automatically** after the main Leaflet workflow completes
2. **Can be triggered manually** via GitHub Actions UI
3. **Runs hourly** as a backup (15 minutes past each hour)
4. **Deploys only the Mapbox HTML** using `keep_files: true` to preserve Leaflet files
5. **Uses the same data** - GeoJSON files already deployed by Leaflet workflow

## Comparison: Leaflet vs Mapbox

| Feature | Leaflet | Mapbox GL |
|---------|---------|-----------|
| **License** | Open source (BSD) | Proprietary (free tier available) |
| **API Key** | Not required | Required (free up to 200k loads/month) |
| **Rendering** | Raster tiles | Vector tiles |
| **Performance** | Good for moderate datasets | Better for large datasets |
| **Visual Quality** | Standard | High quality, smooth animations |
| **Map Styles** | OSM tiles | Multiple professional styles |
| **3D Support** | Limited | Full 3D buildings/terrain |

## Troubleshooting

**Map doesn't load:**
- Check if you've added your Mapbox token
- Check browser console for API errors
- Verify token has correct scopes

**Data not showing:**
- Both versions use identical data sources from `total/` and `daily/` directories
- If Leaflet works but Mapbox doesn't, it's a token issue
- If neither works, it's a data generation issue

**Geocoder not working:**
- Mapbox geocoder requires `geocoding:read` scope
- Check token permissions in Mapbox account
